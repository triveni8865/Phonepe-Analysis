📊 PhonePe Transaction Analysis Dashboard (Power BI)
Project Overview

This Power BI dashboard provides a comprehensive analysis of PhonePe transaction data for the year 2024. The dashboard helps monitor transaction performance, service-wise revenue, payment success rates, and failure reasons across various financial services offered on the platform.


🛠️ Tools & Technologies Used
Power BI Desktop
Data Modeling
Data Visualization

📂 Dataset Information

The dataset contains PhonePe transaction records with the following information:

Transaction ID
Transaction Date
Transaction Status
Service Category
Transaction Amount
Payment Failure Reason
User Information

Key Insights
1. Overall Transaction Performance:
The platform processed a total transaction amount of ₹3,333M during 2024.
A total of 300K transactions were recorded.
The majority of transactions were completed successfully.
2. Service-wise Revenue Analysis:
Loans generated the highest transaction amount (₹102M).
Insurance was the second-highest contributing service (₹22M).
Money Transfer contributed ₹15M.
Recharge Bills generated the lowest transaction value (₹2M).
3. Monthly Transaction Trends:
Transaction amounts fluctuated throughout the year.
August recorded the highest transaction amount (₹13.7M).
September showed a sharp decline (₹10.65M).
Transaction volumes recovered and remained stable in the last quarter.
4. Failed Transaction Analysis:
Server Errors were the leading cause of transaction failures.
Wrong PIN entries were the second most common reason.
Other reasons included:
Insufficient Balance
Wrong Information
Bank Denied
5. Product/Service Performance:
Gold Loan generated the highest amount among individual services.
Bike Loan and Mutual Fund services also showed strong performance.
Utility services such as:
Mobile Recharge
DTH
FASTag Recharge
generated comparatively lower transaction values.
6. Business Recommendations:
Focus on expanding loan-related services as they contribute the most revenue.
Reduce server-related issues to improve transaction success rates.
Educate users about secure PIN usage to decrease failed transactions.
Promote underperforming services through targeted marketing campaigns.
Analyze peak months to understand factors driving higher transaction activity.
